1. What is AWS IAM?

AWS IAM is a service that helps you securely manage access to AWS resources.

2. IAM Key Concepts
  • Users: Individual identities with long-term credentials.
  • Groups: Collection of IAM users for easier policy management.
  • Roles: Used for granting temporary permissions.
  • Policies: JSON documents defining permissions.
  • Identity Providers (IdP): Used for federated authentication.
  • IAM Credentials: Passwords, access keys, security tokens.
3. IAM Policies
  • Written in JSON.
  • Types:
    • Managed Policies: AWS-managed or customer-managed.
    • Inline Policies: Directly attached to a user, group, or role.
    • Service Control Policies (SCPs): Used in AWS Organizations.
    • Permissions Boundaries: Limit the maximum permissions an entity can have.
    • Session Policies: Temporary permissions assigned to a session.
4. IAM Roles
  • Used for granting permissions without long-term credentials.
  • Common use cases:
    • EC2 Instance Role: Grant AWS access to EC2.
    • Cross-Account Access: Assume a role in another AWS account.
    • AWS Services Role: AWS Lambda, ECS, etc.
    • Federated Access: Using SSO or external IdPs.
5. IAM Best Practices
  • ✅ Enable MFA for users, especially root users.
  • ✅ Use IAM roles instead of access keys.
  • ✅ Grant least privilege (principle of least privilege).
  • ✅ Rotate credentials regularly.
