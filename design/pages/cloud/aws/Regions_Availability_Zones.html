Question 1: A company is designing a multi-tier web application that must be highly available and resilient. Which of the following strategies should they implement?

  • A) Deploy all resources in a single Availability Zone within one AWS Region.
  • B) Distribute resources across multiple Availability Zones within a single AWS Region.
  • C) Deploy resources across multiple AWS Regions without using multiple Availability Zones.
  • D) Use edge locations exclusively to host the application.
Answer: B Explanation Distributing resources across multiple Availability Zones within a single AWS Region enhances availability and fault tolerance. This setup ensures that if one AZ experiences issues, the application remains operational through resources in other AZs.

Question 2: Which statement best describes an AWS Availability Zone?

  • A) A fully isolated partition of the AWS infrastructure with its own power, cooling, and networking.
  • B) A collection of data centers spread across multiple geographic locations.
  • C) A global network of edge locations for content delivery.
  • D) The physical location where AWS stores customer data backups.
Answer: A Explanation An Availability Zone (AZ) is a fully isolated partition of the AWS infrastructure within a Region. Each AZ has independent power, cooling, and networking to ensure high availability.

Question 3: A business requires its application to be resilient against regional failures. What is the most effective AWS deployment strategy to achieve this?

  • A) Deploy the application across multiple Availability Zones within a single Region.
  • B) Deploy the application across multiple Regions.
  • C) Use AWS Outposts to host the application on-premises.
  • D) Implement the application using a single Availability Zone and take regular snapshots.
Answer: B Explanation Deploying the application across multiple AWS Regions provides resilience against regional failures, ensuring that if one Region becomes unavailable, the application can continue to operate in another Region.

Question 4: How does AWS define a Region?

  • A) A single data center located in a specific geographic area.
  • B) A collection of edge locations used for content delivery.
  • C) A geographic area containing multiple Availability Zones.
  • D) A specific location for storing backup data.
Answer: C Explanation An AWS Region is a geographic area that consists of multiple, isolated, and physically separate Availability Zones. This design allows for fault tolerance and low-latency connections between AZs within the same Region.

Question 6: Which of the following is a key benefit of using multiple Availability Zones within a Region?

  • A) Lower cost of compute resources
  • B) Reduced network latency across Regions
  • C) Higher availability and fault tolerance
  • D) Automatic scaling of instances
Answer: C Explanation: Using mltiple AZs within a Region provides fault tolerance by distributing workloads across separate physical locations. This minimizes downtime in case of failures.

Question 7: A company is running a web application in a single AWS Region with multiple Availability Zones. What failure scenario can this setup withstand?

  • A) The entire AWS Region going offline
  • B) An Availability Zone failure within the Region
  • C) A global network outage
  • D) A complete failure of Amazon S3
Answer: B Explanation: If an A fails, workloads can be shifted to another AZ within the same Region to maintain availability. However, a Region-wide failure would require multi-Region redundancy.

Question 8: Which AWS service enables high availability by automatically distributing traffic across multiple AZs?

  • B) Amazon CloudFront
  • C) AWS Auto Scaling
  • D) Elastic Load Balancing (ELB)
Answer: D Explanation: ElasticLoad Balancing (ELB) automatically distributes incoming application traffic across multiple targets (EC2 instances, containers) in different AZs, ensuring high availability.

Question 10: Which AWS services replicate data automatically across multiple Availability Zones for fault tolerance? (Choose TWO)

  • A) Amazon S3
  • B) Amazon RDS (when using Multi-AZ deployment)
